# Closet DNA 待解决问题日志

本文件记录尚未完成的问题、产品功能、体验优化、技术债和风险。条目按优先级推进，但优先级不能替代明确的完成标准。

## 状态定义

- 待澄清：方向已提出，但关键需求尚未确定。
- 已定义：需求和完成标准已经明确。
- 设计中：正在制作或评审方案。
- 可开发：设计和依赖已经确认，可以进入实施计划。
- 开发中：正在实现。
- 待验证：实现完成，等待验收。
- 已解决：完成标准已经验证。
- 暂缓：不属于当前阶段，但保留记录。

## 优先级定义

- P0：阻塞当前核心流程或涉及高风险安全问题。
- P1：当前阶段必须完成的重要能力。
- P2：有明确价值，但不阻塞当前阶段。
- P3：探索性方向或长期优化。

## B-20260723-008 提供网购候选商品决策建议

| 字段 | 内容 |
| --- | --- |
| 类型 | 新功能 |
| 优先级 | P3 |
| 状态 | 暂缓 |
| 来源 | 产品愿景第二阶段 |
| 依赖 | 稳定的衣橱数据、风格偏好和商品图处理能力 |

### 完成标准

- 用户可以提交多个候选商品并指定购买场景。
- 系统从与现有衣橱的可搭配度、重复度、场合和偏好给出比较。
- 建议提供理由和不确定性，不替用户虚构尺码或真实上身效果。
- 用户可以记录最终决定并反馈建议是否有帮助。
