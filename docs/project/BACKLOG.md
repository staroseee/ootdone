# OOTDone 待解决问题日志

本文件记录尚未完成的问题、产品功能、体验优化、技术债和风险。条目按优先级推进，但优先级不能替代明确的完成标准。

## 状态定义

- 待澄清：方向已提出，但关键需求尚未确定。
- 已定义：需求和完成标准已经明确。
- 设计中：正在制作或评审方案。
- 可开发：设计和依赖已经确认，可以进入实施计划。
- 开发中：正在实现。
- 待验证：实现完成，等待验收。
- 已解决：完成标准已经验证。
- 暂缓：不属于当前阶段，但保留记录。

## 优先级定义

- P0：阻塞当前核心流程或涉及高风险安全问题。
- P1：当前阶段必须完成的重要能力。
- P2：有明确价值，但不阻塞当前阶段。
- P3：探索性方向或长期优化。

## B-20260723-009 处理外套敞开与叠穿预览

| 字段 | 内容 |
| --- | --- |
| 类型 | 产品 / 图像处理 |
| 优先级 | P0 |
| 状态 | 设计中 |
| 来源 | 用户评审搭配创建流程时发现 |
| 依赖 | B-20260723-002、真人换装能力边界 |
| 相关问题 | P-20260723-003 |

### 问题或机会

完整外套抠图直接覆盖在内搭上，无法表达敞开穿着，还会遮住本应可见的区域。该问题会影响搭配板可信度和真人换装输入设计。

### 预期价值

让用户清楚区分“选品搭配板”和“真人上身模拟”，并在支持外套叠穿时避免生成明显错误或具有误导性的结果。

### 候选方向

已确认采用“搭配板保持平铺编排，真人叠穿交给异步换装”的分层方案。搭配记录外套敞开或扣合状态，并允许补充状态参考图；二维外套前片分割作为后续实验，不默认承诺稳定支持。

### 完成标准

- 搭配板和真人换装的效果承诺明确区分。
- 敞开、扣合等穿着方式能够被记录并传给换装任务。
- 搭配板不使用完整外套图片制造错误的伪上身叠加。
- 支持范围外的图片或穿着方式有明确说明和恢复路径。
- 用户确认方案后同步更新 PRD 与信息架构。

## B-20260723-010 确定产品名称与工程项目名

| 字段 | 内容 |
| --- | --- |
| 类型 | 品牌 / 产品 |
| 优先级 | P1 |
| 状态 | 待验证 |
| 来源 | 用户在信息架构阶段提出项目更名 |
| 依赖 | 微信小程序名称占用和商标正式核验 |

### 问题或机会

原名称 Closet DNA 强调衣橱与个人风格，但没有直接传达“每天穿什么”的核心场景。用户希望小程序产品名称与 OOTD 相关，同时避免被误解为只发布照片的穿搭社区。产品展示名与工程项目名需要分层处理，不能默认必须完全相同。

### 预期价值

让名称更容易关联每日穿搭、个人衣橱和穿搭计划，并为后续视觉设计提供清晰的品牌气质。

### 最终选择

用户于 2026-07-23 选定：

- 中文产品名：`今日搭`
- 英文品牌名：`OOTDone`
- 工程项目名：`ootdone`

“今日搭”直接说明每日穿搭场景；“OOTDone”由 OOTD 与 done 组合，表达快速完成“今天穿什么”的决策。工程项目名遵循简短、小写的英文命名方式。

### 核验记录

- 第一轮公开网络与 GitHub 初筛未发现 `OOTDone` 或“今日搭”作为同类产品的明显直接冲突。
- 初筛只用于排除明显冲突，不证明名称可以注册，也不替代微信小程序名称占用、商标和域名的正式核验。
- PRD 和项目日志已使用新名称。
- 用户已确认统一改名；GitHub 仓库已从 `staroseee/closet-dna` 迁移为 `staroseee/ootdone`。由于当前 Codex 工作区锁定旧目录，已创建并验证 `D:\Projects\ootdone`，旧目录需在切换工作区后删除。

第一轮曾考虑但未采用 `今日衣选 / Ootura / ootura` 和 `今日衣橱 / Dayrobe / dayrobe`；`OOTDly`、`WearCue` 和“今天穿啥”因发现已有公开使用而排除。

### 完成标准

- 名称能够关联 OOTD 或每日穿搭，但不把产品限制为公开内容社区。
- 名称易读、易记，适合微信小程序展示。
- 完成明显重名和公开使用冲突检查。
- 用户明确批准中文展示名、英文品牌名及对应仓库命名方式。
- PRD、项目日志和仓库说明同步更新。

### 剩余工作

- 核验微信小程序名称是否可用。
- 在正式发布前完成目标类别的商标检索。
- 切换到 `D:\Projects\ootdone` 后删除旧目录 `D:\Projects\closet-dna`，完成本地迁移收尾。
